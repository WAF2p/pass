# Regulatory Framework Logos

Drop logo images here to have them appear in the PDF Regulatory Alignment section.

## Naming convention

File name must match the framework name from the YAML control files, lowercased with
spaces, slashes, colons, and parentheses replaced by underscores.

| Framework (as in YAML)          | Expected filename               |
|---------------------------------|---------------------------------|
| GDPR                            | `gdpr.png`                      |
| BSI C5:2020                     | `bsi_c5_2020.png`               |
| ISO 27001:2022                  | `iso_27001_2022.png`            |
| EUCS (ENISA)                    | `eucs__enisa_.png`              |
| AWS Well-Architected Framework  | `aws_well-architected_framework.png` |
| SOC 2                           | `soc_2.png`                     |
| PCI DSS v4.0                    | `pci_dss_v4_0.png`              |
| NIST SP 800-161                 | `nist_sp_800-161.png`           |
| FinOps Foundation               | `finops_foundation.png`         |
| ITIL 4                          | `itil_4.png`                    |

## Supported formats

PNG, JPG/JPEG. Recommended size: at least 128 × 128 px, square or near-square.

If no file is found, the PDF renders a colored initials badge as fallback.
